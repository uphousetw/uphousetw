🧩 核心組件架構

App.tsx

src/App.tsx
應用程式根組件，配置路由系統和 GA4 初始化。包含所有主要路由定義和 SEO 組件整合。

Layout.tsx

src/components/Layout.tsx
頁面布局組件，提供統一的頁面結構，包含 Header、主要內容區域和 Footer。

Header.tsx

src/components/Header.tsx
網站導航頭部，包含 Logo、主選單和響應式行動選單功能。

Footer.tsx

src/components/Footer.tsx
網站底部，包含公司資訊、聯絡方式和法規連結（隱私權政策、免責聲明）。

SEO.tsx

src/components/SEO.tsx
SEO 優化組件，動態設定每個頁面的 meta title、description 和其他 SEO 標籤。

Home.tsx

src/pages/Home.tsx
首頁組件，包含 Hero 區塊、服務介紹、精選建案展示和 CTA 呼籲行動。

Projects.tsx

src/pages/Projects.tsx
建案列表頁面，提供建案篩選（類別、年份）和排序功能，支援響應式網格布局。

ProjectDetail.tsx

src/pages/ProjectDetail.tsx
建案詳情頁面，顯示建案完整資訊包含封面圖、詳細資料和內文描述。

Contact.tsx

src/pages/Contact.tsx
聯絡表單頁面，包含預設值設定、表單驗證和送出後的成功提示訊息。

Admin.tsx

src/pages/Admin.tsx
後台管理系統，包含登入驗證、內容管理、部署控制和聯絡訊息查看功能。

{ "name": "王大名", // 聯絡人姓名（必填） "phone": "0932-123456", // 聯絡電話（必填） "email": "example@mail.com", // Email（選填） "project": "電梯大樓", // 感興趣的建案類型（必填） "message": "詢問內容...", // 訊息內容（必填） "createdAt": "2025-09-25T08:00:00Z" // 建立時間 }

{ "id": "user-uuid", // 使用者唯一 ID "email": "admin@company.com", // Email 地址 "role": "admin", // 角色：admin/editor "name": "管理員" // 顯示名稱 }

🔒 安全性設計

JWT 認證 - 使用 HttpOnly Cookie 儲存 JWT Token，防止 XSS 攻擊
HTTPS 強制加密 - 所有資料傳輸都通過 HTTPS 加密保護
CORS 設定 - 限制跨來源請求，保護 API 端點安全
管理員白名單 - 只有預設的 Email 地址才能存取後台管理功能
檔案上傳限制 - 限制檔案類型、大小和上傳頻率
PII 資料保護 - 個人識別資訊不會傳送至 Google Analytics
敏感路徑保護 - 管理員相關檔案和配置受到存取限制

📈 Google Analytics 4 整合

前台事件追蹤

  • page_view - 頁面瀏覽追蹤
  • contact_form_submit - 聯絡表單提交
  • project_view - 建案詳情查看
  • navigation_click - 導航點擊追蹤

後台事件追蹤

  • admin_login - 管理員登入（參數：method, result）
  • admin_logout - 管理員登出
  • admin_action - CRUD 操作（參數：action, entity, status）
  • admin_upload - 檔案上傳（參數：file_type, size_kb, status）
  • admin_deploy_triggered - 部署觸發
  • admin_deploy_result - 部署結果（參數：status, build_id）
  • admin_error - 錯誤記錄（參數：where, code, message）

🚀 部署架構

Netlify 自動部署流程

1. Git Push - 程式碼推送到 GitHub repository
2. 自動觸發 - Netlify 偵測到程式碼變更自動開始建構
3. 依賴安裝 - 執行 npm install 安裝所有依賴套件
4. 建構程序 - 執行 npm run build 建構生產版本
5. 靜態檔案部署 - 將 dist/ 資料夾內容部署到 CDN
6. Serverless Functions - 部署 netlify/functions/ 中的 API 端點
7. 部署完成 - 新版本上線，發送通知

環境變數配置

REACT_APP_SITE_URL=https://your-site.netlify.app REACT_APP_GA_MEASUREMENT_ID=G-XXXXXXXXXX JWT_SECRET=your-secure-secret-key ADMIN_EMAILS=admin@company.com,manager@company.com

⚡ 效能優化

前端優化

  • 圖片延遲載入 - 使用 Intersection Observer API 實現圖片 lazy loading
  • 程式碼分割 - React.lazy() 和 Suspense 實現路由層級的程式碼分割
  • 快取策略 - 靜態資源設定適當的 Cache-Control 標頭
  • 壓縮優化 - Gzip/Brotli 壓縮減少傳輸大小
  • CDN 加速 - Netlify 全球 CDN 提供快速內容傳送

SEO 優化

  • Meta 標籤 - 動態生成每頁的 title 和 description
  • 結構化資料 - JSON-LD 格式的 Organization 和 Project 結構化資料
  • Sitemap - 自動生成並更新的 XML sitemap
  • Robots.txt - 爬蟲友善的 robots.txt 配置
  • 語義化 HTML - 使用適當的 HTML5 語義標籤

📋 法規合規

隱私權政策

  • 個資蒐集範圍 - 明確說明蒐集的個人資料類型
  • 使用目的 - 詳細說明個資使用目的和範圍
  • 保存期限 - 自收到日起 5 年，期滿即刪除
  • 安全措施 - HTTPS 加密傳輸和存取權限控制
  • 使用者權利 - 提供查詢、更正、刪除個資的管道

Cookie 政策

  • 必要 Cookie - 僅使用維持網站運作必需的 Cookie
  • 分析 Cookie - Google Analytics 4 匿名分析
  • 驗證 Cookie - 後台登入使用 HttpOnly Cookie

免責聲明

  • 內容準確性 - 網站內容僅供參考，不構成法律承諾
  • 服務變更 - 保留隨時修改網站內容的權利
  • 責任限制 - 明確規範公司責任範圍

📞 技術支援與聯絡

疑難排解

建構錯誤

清除 node_modules 並重新安裝：
rm -rf node_modules package-lock.json
npm install

樣式問題

重新建構 CSS：
npm run build
檢查 Tailwind 配置是否正確

部署失敗

檢查 Netlify 環境變數設定
確認 Build 指令和輸出目錄設定
查看 Netlify Functions 日誌

後台存取問題

確認 Email 在管理員白名單中
檢查 JWT_SECRET 環境變數
清除瀏覽器 Cookie 重新登入
